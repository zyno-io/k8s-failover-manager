k8s-failover-manager is a Kubebuilder/controller-runtime Kubernetes operator that manages DNS-based failover across two clusters. Primary entrypoints are cmd/main.go for the controller-manager and cmd/connection-killer/main.go for the DaemonSet agent. Core code lives in internal/controller (state machine), internal/action (action executors), internal/remotecluster (sync), internal/connkill (connection flush signaling/agent), internal/service (managed Service/EndpointSlice), and internal/metrics.
